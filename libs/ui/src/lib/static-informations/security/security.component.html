DATA SECURITY

Données et documents

Deux types de bases de données Firebase sont utilisées pour Blockframes :

  1. Firestore : base de données NoSQL destinée aux données utilisateurs, sauvegardées quotidiennement sur deux serveurs distincts pour assurer une haute disponibilité et une récupération rapide en cas de panne. Les données peuvent être exportées en format jsonl sur demande, offrant ainsi une flexibilité pour l'analyse et la migration des données.
  2. File Storage : solution de stockage pour documents utilisateurs, tels que des images, des fichiers PDF, Excel, etc. Cette plateforme assure une gestion efficace des fichiers volumineux ou des documents sensibles, facilitant leur partage et leur accès sécurisé.

Sécurité

La sécurité des données et des documents sur Firebase est notre priorité absolue. Voici les mesures de sécurité clés mises en place :

  • Chiffrement des données : Firestore et File Storage utilisent un chiffrement robuste en transit et au repos. En transit, TLS (Transport Layer Security) assure la sécurité des données lors de leur transfert entre le client et Firebase. Au repos, les données sont chiffrées à l'aide de techniques avancées qui garantissent leur protection contre les accès non autorisés.
  • Authentification : L’accès aux bases de données est strictement contrôlé. Par défaut, seuls les utilisateurs authentifiés peuvent accéder aux données, grâce à un système d’authentification par mail et mot de passe. Ce service Firebase assure le chiffrement de bout en bout des mots de passe, empêchant même les administrateurs Cascade8 d'y accéder.
  • Règles d'accès : Des règles d'accès détaillées sont définies pour chaque type de document ou de donnée. Ces règles garantissent que seules les personnes autorisées, comme les utilisateurs invités par le propriétaire d'un document ou les administrateurs Cascade8 en cas de maintenance, peuvent accéder aux informations sensibles, telles que les conditions financières d'un projet.

Certifications ISO de sécurité et de chiffrement

Pour soutenir notre engagement envers la sécurité, Google Cloud Platform, qui héberge Firebase, a obtenu plusieurs certifications ISO clés, renforçant ainsi la confiance dans nos solutions de sécurité et de chiffrement :

  • ISO/IEC 27001 : Cette norme est l'une des plus reconnues mondialement pour le système de gestion de la sécurité de l'information (SGSI). Elle certifie que Google Cloud Platform dispose de processus rigoureux pour gérer et protéger les données des utilisateurs.
  • ISO/IEC 27017 : Spécifique au cloud, cette certification concerne la sécurité des informations et fournit des directives sur les aspects de la sécurité de l'information à gérer par un fournisseur de services cloud.
  • ISO/IEC 27018 : Axée sur la protection des données personnelles dans le cloud, elle garantit que Google Cloud adhère aux pratiques de confidentialité reconnues internationalement, offrant ainsi une protection supplémentaire pour les données personnelles stockées sur Firebase.

En intégrant ces normes internationales à notre infrastructure et à nos processus, nous assurons une protection maximale des données et des documents de Blockframes, tout en respectant les exigences réglementaires et en renforçant la confiance des utilisateurs dans notre plateforme.

Datos y documentos

Para Blockframes se utilizan dos tipos de bases de datos de Firebase:

  1. Firestore : Una base de datos NoSQL diseñada para datos de usuario, respaldada diariamente en dos servidores separados para asegurar alta disponibilidad y recuperación rápida en caso de falla. Los datos pueden ser exportados en formato json bajo solicitud, proporcionando flexibilidad para el análisis y la migración de datos.
  2. Almacenamiento de Archivos : Una solución de almacenamiento para documentos de usuarios, como imágenes, archivos PDF, Excel, etc. Esta plataforma asegura una gestión eficiente de archivos grandes o documentos sensibles, facilitando su compartición y acceso seguro.

Seguridad

La seguridad de los datos y documentos en Firebase es nuestra máxima prioridad. Aquí están las principales medidas de seguridad implementadas:

  • Cifrado de Datos : Firestore y Almacenamiento de Archivos usan un cifrado robusto tanto en tránsito como en reposo. En tránsito, TLS (Transport Layer Security) asegura la seguridad de los datos durante la transferencia entre el cliente y Firebase. En reposo, los datos están cifrados usando técnicas avanzadas que garantizan la protección contra el acceso no autorizado.
  • Autenticación : El acceso a las bases de datos está estrictamente controlado. Por defecto, solo los usuarios autenticados pueden acceder a los datos, utilizando un sistema de autenticación por correo electrónico y contraseña. Este servicio de Firebase asegura el cifrado de extremo a extremo de las contraseñas, evitando que incluso los administradores de Cascade8 puedan acceder a ellas.
  • Reglas de Acceso : Se definen reglas de acceso detalladas para cada tipo de documento o dato. Estas reglas aseguran que solo personas autorizadas, como usuarios invitados por el propietario de un documento o administradores de Cascade8 durante el mantenimiento, puedan acceder a información sensible, como los términos financieros de un proyecto.

Certificaciones ISO de Seguridad y Cifrado

Para respaldar nuestro compromiso con la seguridad, Google Cloud Platform, que hospeda Firebase, ha obtenido varias certificaciones clave ISO, aumentando así la confianza en nuestras soluciones de seguridad y cifrado:

  • ISO/IEC 27001 : Este estándar es uno de los más reconocidos globalmente para Sistemas de Gestión de Seguridad de la Información (SGSI). Certifica que Google Cloud Platform tiene procesos rigurosos para gestionar y proteger los datos de los usuarios.
  • ISO/IEC 27017 : Específico para la nube, esta certificación se enfoca en la seguridad de la información y proporciona directrices sobre los aspectos de seguridad de la información que deben ser gestionados por un proveedor de servicios en la nube.
  • ISO/IEC 27018 : Centrada en la protección de datos personales en la nube, asegura que Google Cloud adhiera a prácticas de privacidad reconocidas internacionalmente, proporcionando así una protección adicional para los datos personales almacenados en Firebase.

Al integrar estos estándares internacionales en nuestra infraestructura y procesos, garantizamos la máxima protección de los datos y documentos de Blockframes, cumpliendo además con los requisitos regulatorios y mejorando la confianza de los usuarios en nuestra plataforma.

Data and documents

Two types of Firebase databases are used for Blockframes:

Security

The security of data and documents on Firebase is our top priority. Here are the key security measures in place:

  • Data Encryption : Firestore and File Storage use robust encryption both in transit and at rest. In transit, TLS (Transport Layer Security) ensures the security of data during transfer between the client and Firebase.   • Authentication : Access to the databases is strictly controlled. By default, only authenticated users can access the data, using an email and password authentication system. This Firebase service ensures end-to-end encryption of passwords, preventing even Cascade8 administrators from accessing them.
  • Access Rules : Detailed access rules are defined for each type of document or data. These rules ensure that only authorized individuals, such as users invited by the owner of a document or Cascade8 administrators during maintenance, can access sensitive information, such as the financial terms of a project.
