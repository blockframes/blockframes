DATA SECURITY

Données et documents

Deux types de bases de données Firebase sont utilisées pour Blockframes :

  1. Firestore : base de données NoSQL destinée aux données utilisateurs, sauvegardées quotidiennement sur deux serveurs distincts pour assurer une haute disponibilité et une récupération rapide en cas de panne. Les données peuvent être exportées en format jsonl sur demande, offrant ainsi une flexibilité pour l'analyse et la migration des données.
  2. File Storage : solution de stockage pour documents utilisateurs, tels que des images, des fichiers PDF, Excel, etc. Cette plateforme assure une gestion efficace des fichiers volumineux ou des documents sensibles, facilitant leur partage et leur accès sécurisé.

Sécurité

La sécurité des données et des documents sur Firebase est notre priorité absolue. Voici les mesures de sécurité clés mises en place :

  • Chiffrement des données : Firestore et File Storage utilisent un chiffrement robuste en transit et au repos. En transit, TLS (Transport Layer Security) assure la sécurité des données lors de leur transfert entre le client et Firebase. Au repos, les données sont chiffrées à l'aide de techniques avancées qui garantissent leur protection contre les accès non autorisés.
  • Authentification : L’accès aux bases de données est strictement contrôlé. Par défaut, seuls les utilisateurs authentifiés peuvent accéder aux données, grâce à un système d’authentification par mail et mot de passe. Ce service Firebase assure le chiffrement de bout en bout des mots de passe, empêchant même les administrateurs Cascade8 d'y accéder.
  • Règles d'accès : Des règles d'accès détaillées sont définies pour chaque type de document ou de donnée. Ces règles garantissent que seules les personnes autorisées, comme les utilisateurs invités par le propriétaire d'un document ou les administrateurs Cascade8 en cas de maintenance, peuvent accéder aux informations sensibles, telles que les conditions financières d'un projet.

Certifications ISO de sécurité et de chiffrement

Pour soutenir notre engagement envers la sécurité, Google Cloud Platform, qui héberge Firebase, a obtenu plusieurs certifications ISO clés, renforçant ainsi la confiance dans nos solutions de sécurité et de chiffrement :

  • ISO/IEC 27001 : Cette norme est l'une des plus reconnues mondialement pour le système de gestion de la sécurité de l'information (SGSI). Elle certifie que Google Cloud Platform dispose de processus rigoureux pour gérer et protéger les données des utilisateurs.
  • ISO/IEC 27017 : Spécifique au cloud, cette certification concerne la sécurité des informations et fournit des directives sur les aspects de la sécurité de l'information à gérer par un fournisseur de services cloud.
  • ISO/IEC 27018 : Axée sur la protection des données personnelles dans le cloud, elle garantit que Google Cloud adhère aux pratiques de confidentialité reconnues internationalement, offrant ainsi une protection supplémentaire pour les données personnelles stockées sur Firebase.

En intégrant ces normes internationales à notre infrastructure et à nos processus, nous assurons une protection maximale des données et des documents de Blockframes, tout en respectant les exigences réglementaires et en renforçant la confiance des utilisateurs dans notre plateforme.

Data and documents

Two types of Firebase databases are used for Blockframes:

  1. Firestore : A NoSQL database designed for user data, backed up daily on two separate servers to ensure high availability and quick recovery in case of failure. Data can be exported in json format upon request, providing flexibility for data analysis and migration.
  2. File Storage : A storage solution for user documents, such as images, PDF files, Excel, etc. This platform ensures efficient management of large files or sensitive documents, facilitating their sharing and secure access.

Security

The security of data and documents on Firebase is our top priority. Here are the key security measures in place:

  • Data Encryption : Firestore and File Storage use robust encryption both in transit and at rest. In transit, TLS (Transport Layer Security) ensures the security of data during transfer between the client and Firebase. At rest, data is encrypted using advanced techniques that ensure protection against unauthorized access.
  • Authentication : Access to the databases is strictly controlled. By default, only authenticated users can access the data, using an email and password authentication system. This Firebase service ensures end-to-end encryption of passwords, preventing even Cascade8 administrators from accessing them.
  • Access Rules : Detailed access rules are defined for each type of document or data. These rules ensure that only authorized individuals, such as users invited by the owner of a document or Cascade8 administrators during maintenance, can access sensitive information, such as the financial terms of a project.

Security and Encryption ISO Certifications

To support our commitment to security, Google Cloud Platform, which hosts Firebase, has obtained several key ISO certifications, thereby enhancing confidence in our security and encryption solutions:

  • ISO/IEC 27001 : This standard is one of the most globally recognized for Information Security Management Systems (ISMS). It certifies that Google Cloud Platform has rigorous processes in place to manage and protect user data.
  • ISO/IEC 27017 : Specific to the cloud, this certification focuses on information security and provides guidelines on information security aspects to be managed by a cloud service provider.
  • ISO/IEC 27018 : Focused on the protection of personal data in the cloud, it ensures that Google Cloud adheres to internationally recognized privacy practices, thus providing additional protection for personal data stored on Firebase.

By integrating these international standards into our infrastructure and processes, we ensure maximum protection of Blockframes' data and documents while also complying with regulatory requirements and enhancing users' trust in our platform.
